The following is a set of guidelines for contributing to Dockwatch Agent. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs

- **Search for existing issues**: Check if the bug has already been reported.
- **Open a new issue**: Clearly describe the bug and include steps to reproduce it.

### Suggesting Enhancements

- **Open a new issue**: Explain why the enhancement would be useful and how it should work.

### Pull Requests

1. Fork the repo and create your branch from `main`.
2. Install dependencies: `npm install`.
3. If you've added code that should be tested, add tests.
4. Ensure the test suite passes.
5. Provide a clear description of the change in your PR.

## Styleguides

### Git Commit Messages

- Use the present tense ("Add feature" not "Added feature").
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
- Limit the first line to 72 characters or less.

### JavaScript Styleguide

- Use 2 spaces for indentation.
- Use semicolons.
- Follow standard Node.js patterns.

## Code of Conduct

Help us keep Dockwatch Agent open and inclusive. Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md).
